# Ставь Угольки

Проект разделён на независимые части:

- `apps/shop` — клиентская витрина для Telegram Mini App и установки на телефон
- `apps/owner` — кабинет владельца
- `bot` — отдельный Telegram-бот
- `server` — лёгкий API и раздача статики без npm-зависимостей
- `data` — товары, баннеры, заказы, контакты, история постов
- `apps/shared` — общие стили, логика, assets

## Запуск

```bash
cp .env.example .env
npm start
```

Откроется:

- магазин: `http://localhost:3000/shop/`
- владелец: `http://localhost:3000/owner/`
- короткий адрес владельца: `http://localhost:3000/admin/`

## Что есть в этой версии

- витрина в тёмно-tiffany стиле
- быстрые фильтры в порядке: табак → уголь → кальяны → прочее
- у баннеров вместо внешней ссылки — подборка товаров по фильтрам
- фильтры по категории, цене, бренду
- для табака — выбор граммовки на карточке и в карточке товара
- подфильтры брендов для табака
- подробная карточка товара с текстом под фото
- корзина со счётчиком и анимацией падения товара
- оформление заказа с телефоном или ссылкой на Telegram
- отправка заказа в Telegram-группу, когда будет задан `BOT_TOKEN` и `ADMIN_GROUP_CHAT_ID`
- поддержка редактируемых контактов
- редактор товаров, баннеров и постов в кабинете владельца
- отправка поста в канал/группу от имени бота с картинкой
- PWA: магазин можно установить на телефон

## Telegram

Бот запускается отдельно:

```bash
node bot/index.js
```

Для работы магазина через Telegram и отправки сообщений бота нужны переменные:

- `BOT_TOKEN`
- `MINIAPP_URL`
- `ADMIN_GROUP_CHAT_ID`
- `CHANNEL_CHAT_ID`

## Где редактировать дальше

- визуал магазина: `apps/shop/css/shop.css`
- логика магазина: `apps/shop/js/shop.js`
- визуал владельца: `apps/owner/css/owner.css`
- логика владельца: `apps/owner/js/owner.js`
- сервер и API: `server/index.js`
- бот: `bot/index.js`
- товары: `data/products.json`
- баннеры: `data/banners.json`

## Примечание

Посты и заказы в Telegram начнут реально отправляться после того, как вы добавите значения в `.env`:

- `BOT_TOKEN`
- `ADMIN_GROUP_CHAT_ID`
- `CHANNEL_CHAT_ID`

До этого магазин, кабинет владельца и PWA всё равно будут работать локально.

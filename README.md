# Ставь Угольки — Telegram Mini App Shop

Проект разделён на три части:
- `apps/shop` — клиентский магазин
- `apps/owner` — страница владельца
- `bot` — Telegram-бот
- `server` — API и раздача статики

## Локальный запуск

```bash
cp .env.example .env
npm start
```

Открыть:
- магазин: `http://localhost:3000/shop/`
- владелец: `http://localhost:3000/owner/`
- также работает `http://localhost:3000/admin/`

Логин владельца по умолчанию:
- логин: `owner`
- пароль: `stavugolki2026`

## Как сделать, чтобы данные не сбрасывались на Railway

Railway пересобирает контейнер при новом деплое. Если хранить JSON внутри проекта, изменения будут сбрасываться. Поэтому проект умеет работать с **Volume**.

### Что сделать в Railway

1. Создать `Volume`.
2. Подключить его к сервису с приложением.
3. Указать mount path: `/data`.
4. В Variables добавить:
   - `DATA_DIR=/data`
5. Сделать redeploy.

После этого:
- товары
- баннеры
- контакты поддержки
- заказы
- посты

будут сохраняться в Volume и **не будут сбрасываться при новом деплое**.

## Как работает миграция

При первом запуске сервер сам копирует стартовые файлы из папки `data/` в `DATA_DIR`, если в Volume ещё пусто.

## Проверка

Проверьте health endpoint:

```bash
GET /api/health
```

Он вернёт текущий `dataDir`. Если там `/data`, значит сохранение настроено правильно.

## Telegram бот

Запустите отдельным процессом:

```bash
node bot/index.js
```

Команды в Telegram:

- `/meneger` или `/manager` — назначить текущую группу группой менеджера для заказов
- `/postgroup` — назначить текущую группу для публикации постов
- `/roles` — показать роли текущей группы
- `/unregister` — убрать роли текущей группы
- `/shop` — открыть магазин

Для кнопки под постом нужен `BOT_USERNAME` или корректный `MINIAPP_URL`.
